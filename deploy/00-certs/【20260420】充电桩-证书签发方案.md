# 充电桩证书签发方案

## 1. 背景

历史测试证书已归档至 `archive/20260420-historical/`，本方案按照 `【202604】充电桩-证书签发架构设计.md` 的设计规范，定义新证书的签发与归档方案。

---

## 2. 证书类型

本方案涉及两类证书：

| 证书类型 | 用途 | 颁发给 | Key Usage | Extended Key Usage |
|---------|------|--------|-----------|-------------------|
| **型号证书** | 充电桩设备身份标识 | 充电桩设备 | Digital Signature, Key Encipherment | TLS Web Client Authentication |
| **服务端证书** | 服务端（如 EMQX）TLS 身份 | EMQX、API Gateway 等服务端 | Digital Signature, Key Encipherment | TLS Web Server Authentication |

## 6. 签发流程（CSR 模式）

> **说明**：采用 CSR 模式签发。业务系统负责生成密钥对和 CSR，CA 服务端仅对 CSR 进行签名，不持有私钥。

### 6.1 根 CA 生成（离线一次性操作）

```bash
# 1. 生成根 CA 私钥（离线环境）
openssl ecparam -name prime256v1 -genkey -out root-ca/private/ca.key

# 2. 生成根 CA 自签名证书（99年有效期）
openssl req -x509 -new -nodes -key root-ca/private/ca.key \
  -sha256 -days 36135 \
  -subj "/C=CN/ST=hunan/L=zhuzhou/O=株洲中车时代电气股份有限公司绿能分公司/CN=ZZCQC Root CA" \
  -out root-ca/certs/root-ca.crt
```

### 6.2 中间 CA 生成

```bash
# 1. 生成中间 CA 私钥
openssl ecparam -name prime256v1 -genkey -out intermediate-ca/private/intermediate-ca.key

# 2. 生成 CSR
openssl req -new -key intermediate-ca/private/intermediate-ca.key \
  -subj "/C=CN/ST=hunan/L=zhuzhou/O=株洲中车时代电气股份有限公司绿能分公司/CN=ZZCQC Intermediate CA" \
  -out intermediate-ca/csr/intermediate-ca.csr

# 3. 使用根 CA 签发中间 CA 证书
openssl x509 -req -in intermediate-ca/csr/intermediate-ca.csr \
  -CA root-ca/certs/root-ca.crt -CAkey root-ca/private/ca.key \
  -CAcreateserial -days 36135 -sha256 \
  -extfile templates/intermediate-ca.cnf \
  -out intermediate-ca/certs/intermediate-ca.crt
```

### 6.3 型号证书签发（CSR 模式）

```bash
#!/bin/bash
# issue-model-cert.sh
# 用法: ./issue-model-cert.sh <CSR文件路径>
# 示例: ./issue-model-cert.sh /path/to/csr.pem

set -e

if [ -z "$1" ]; then
  echo "用法: $0 <CSR文件路径>"
  exit 1
fi

CSR_FILE="$1"
CERTS_DIR="$(cd "$(dirname "$0")/.." && pwd)/certs/models"
CA_DIR="$(cd "$(dirname "$0")/.." && pwd)/ca"
TEMPLATE_DIR="$(cd "$(dirname "$0")/../templates" && pwd)"
MODEL_NAME="CZ-EVSE"

echo "=== 签发型号证书 (CSR 模式) ==="
echo "CSR 文件: ${CSR_FILE}"

mkdir -p "${CERTS_DIR}/${MODEL_NAME}/csr"

# 1. 验证 CSR 合法性
echo "CSR 信息:"
openssl req -in "${CSR_FILE}" -noout -subject

# 2. 使用中间 CA 签发证书
openssl x509 -req \
  -in "${CSR_FILE}" \
  -CA "${CA_DIR}/intermediate-ca/certs/intermediate-ca.crt" \
  -CAkey "${CA_DIR}/intermediate-ca/private/intermediate-ca.key" \
  -CAcreateserial \
  -days 36135 -sha256 \
  -extfile "${TEMPLATE_DIR}/model-cert.cnf" \
  -out "${CERTS_DIR}/${MODEL_NAME}/cert.pem"

# 3. 归档 CSR
DEST_CSR="${CERTS_DIR}/${MODEL_NAME}/csr/$(basename "${CSR_FILE}")"
if [ "${CSR_FILE}" != "${DEST_CSR}" ] && ! cmp -s "${CSR_FILE}" "${DEST_CSR}"; then
  cp "${CSR_FILE}" "${DEST_CSR}"
fi

echo "型号证书签发完成:"
openssl x509 -in "${CERTS_DIR}/${MODEL_NAME}/cert.pem" -noout -subject -dates
echo "证书指纹: $(openssl x509 -in "${CERTS_DIR}/${MODEL_NAME}/cert.pem" -noout -fingerprint -sha256)"
```

### 6.4 服务端证书签发（CSR 模式，动态 SAN）

```bash
#!/bin/bash
# issue-server-cert.sh
# 用法: ./issue-server-cert.sh <CSR文件路径> <CN> [SAN列表...]
# 示例: ./issue-server-cert.sh /path/to/server.csr 127.0.0.1 IP:127.0.0.1
# 示例: ./issue-server-cert.sh /path/to/server.csr emqx.internal DNS:emqx.internal

set -e

if [ -z "$1" ] || [ -z "$2" ]; then
  echo "用法: $0 <CSR文件路径> <CN> [SAN列表...]"
  echo "示例: $0 /path/to/server.csr 127.0.0.1 IP:127.0.0.1"
  exit 1
fi

CSR_FILE="$1"
CN="$2"
shift 2
SAN_LIST=("$@")

CERTS_DIR="$(cd "$(dirname "$0")/.." && pwd)/certs/servers"
CA_DIR="$(cd "$(dirname "$0")/.." && pwd)/ca"

echo "=== 签发服务端证书 (CSR 模式) ==="
echo "CSR 文件: ${CSR_FILE}"
echo "CN:       ${CN}"
[ ${#SAN_LIST[@]} -gt 0 ] && echo "SAN:      ${SAN_LIST[*]}"

mkdir -p "${CERTS_DIR}/csr"

# 1. 验证 CSR 合法性
echo ""
echo "CSR 信息:"
openssl req -in "${CSR_FILE}" -noout -subject

# 2. 动态生成带 SAN 的配置文件
TEMP_CNF=$(mktemp)
cat > "${TEMP_CNF}" << 'TPL'
basicConstraints = CA:FALSE
keyUsage = critical,digitalSignature,keyEncipherment
extendedKeyUsage = critical,serverAuth
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid,issuer
subjectAltName = @alt_names

[alt_names]
TPL

# 追加 SAN 条目
for SAN in "${SAN_LIST[@]}"; do
  KEY=$(echo "$SAN" | cut -d: -f1)
  VAL=$(echo "$SAN" | cut -d: -f2-)
  echo "${KEY} = ${VAL}" >> "${TEMP_CNF}"
done

# 3. 使用中间 CA 签发证书
openssl x509 -req \
  -in "${CSR_FILE}" \
  -CA "${CA_DIR}/intermediate-ca/certs/intermediate-ca.crt" \
  -CAkey "${CA_DIR}/intermediate-ca/private/intermediate-ca.key" \
  -CAcreateserial \
  -days 36135 -sha256 \
  -extfile "${TEMP_CNF}" \
  -out "${CERTS_DIR}/cert.pem"

rm -f "${TEMP_CNF}"

# 4. 归档 CSR
DEST_CSR="${CERTS_DIR}/csr/$(basename "${CSR_FILE}")"
if [ "${CSR_FILE}" != "${DEST_CSR}" ] && ! cmp -s "${CSR_FILE}" "${DEST_CSR}"; then
  cp "${CSR_FILE}" "${DEST_CSR}"
fi

echo ""
echo "服务端证书签发完成:"
openssl x509 -in "${CERTS_DIR}/cert.pem" -noout -subject -dates
echo "SAN: $(openssl x509 -in "${CERTS_DIR}/cert.pem" -noout -text | grep -A1 "X509v3 Subject Alternative Name" | tail -1)"
```

## 7. OpenSSL 模板配置

> **参考历史配置**：历史 `openssl.cnf` 中定义了 `subjectAltName`、`subjectKeyIdentifier` 等扩展字段。本方案模板参照同等配置级别设计。

### 7.1 型号证书模板 (`templates/model-cert.cnf`)

```ini
basicConstraints = CA:FALSE
keyUsage = critical,digitalSignature,keyEncipherment
extendedKeyUsage = critical,clientAuth
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid,issuer
```

### 7.2 服务端证书模板参考 (`templates/server-cert.cnf`)

> **说明**：服务端证书签发时 SAN 由脚本动态生成，此模板仅作扩展字段结构参考。

```ini
basicConstraints = CA:FALSE
keyUsage = critical,digitalSignature,keyEncipherment
extendedKeyUsage = critical,serverAuth
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid,issuer
# subjectAltName 由签发脚本动态填充
```

### 7.3 中间 CA 模板 (`templates/intermediate-ca.cnf`)

```ini
basicConstraints = critical,CA:true,pathlen:0
keyUsage = critical,digitalSignature,cRLSign,keyCertSign
extendedKeyUsage = serverAuth,clientAuth
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid,issuer
```

## 8. 服务端证书申请示例

### 8.1 生成服务端 CSR（业务系统侧）

```bash
#!/bin/bash
# generate-server-csr.sh
# 用法: ./generate-server-csr.sh <CN> [SAN列表...]
# 示例: ./generate-server-csr.sh 127.0.0.1 IP:127.0.0.1
# 示例: ./generate-server-csr.sh emqx.internal DNS:emqx.internal

set -e

CN="$1"
shift
SAN_LIST=("$@")

CERTS_DIR="$(cd "$(dirname "$0")/.." && pwd)/certs/servers"

echo "=== 生成服务端 CSR ==="
echo "CN:   ${CN}"
[ ${#SAN_LIST[@]} -gt 0 ] && echo "SAN:  ${SAN_LIST[*]}"

mkdir -p "${CERTS_DIR}/private" "${CERTS_DIR}/csr"

# 1. 生成私钥
openssl ecparam -name prime256v1 -genkey -out "${CERTS_DIR}/private/server.key"
chmod 600 "${CERTS_DIR}/private/server.key"

# 2. 生成 CSR
openssl req -new \
  -key "${CERTS_DIR}/private/server.key" \
  -subj "/C=CN/ST=hunan/L=zhuzhou/O=株洲中车时代电气股份有限公司绿能分公司/CN=${CN}" \
  -out "${CERTS_DIR}/csr/server.csr"

echo ""
echo "CSR 生成完成:"
echo "  私钥: ${CERTS_DIR}/private/server.key"
echo "  CSR:  ${CERTS_DIR}/csr/server.csr"
openssl req -in "${CERTS_DIR}/csr/server.csr" -noout -subject
echo ""
echo "请将 CSR 文件和 SAN 配置提交给 CA 签发，私钥请妥善保管。"
```

## 9. 归档规则

| 证书类型 | 归档位置 | 说明 |
|---------|---------|------|
| 根 CA | `ca/root-ca/private/` | 加密存储，离线管理 |
| 中间 CA | `ca/intermediate-ca/` | 在线签发服务器 |
| 型号证书 | `certs/models/CZ-EVSE/` | 通用型号证书，CSR 归档在 `csr/` 子目录 |
| 服务端证书 | `certs/servers/` | 统一目录，CSR 归档在 `csr/` 子目录 |

**按时间归档**: 历史证书移入 `archive/YYYYMMDD-描述/` 目录

> **说明**：证书本身是唯一真实来源，所有证书信息均可通过 `openssl x509 -text` 读取，无需维护额外的元数据文件。
